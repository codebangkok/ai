
# Project Context

## Architecture Overview
This is a microservices application using:
- Node.js with Express for API services
- React with TypeScript for frontend
- PostgreSQL with Prisma ORM
- Redis for caching

## Coding Standards
- Use functional components with hooks
- Implement error boundaries for all route components
- Follow RESTful API conventions
- Write unit tests for all business logic


# Personal Development Preferences

## Code Style
- Always use explicit return types in TypeScript
- Prefer const assertions over type annotations
- Use descriptive variable names, avoid abbreviations

## Workflow Shortcuts
- When writing tests, use Jest with React Testing Library
- Always run “npm run lint’ before commits
- Prefer composition over inheritance


## When using the Edit tool:
- Always check existing code patterns before making changes
- Maintain consistent indentation and style
- Look for existing utility functions before creating new ones

## When using the Bash tool:
- Always check if package.json scripts exist before running raw commands
- Validate file paths exist before operations
- Use relative paths when possible
